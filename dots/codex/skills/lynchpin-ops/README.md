# Lynchpin Ops Skill

This skill is not just a command index.

It is the evolving operator doctrine for using Lynchpin to build
multi-scale personal understanding from local data and to assemble
LLM-usable context from that substrate.

## Scope

Use this skill when the task is any of:

- validating or refreshing Lynchpin data planes,
- deriving higher-level understanding from raw signals,
- improving project/topic/activity attribution,
- building trajectory artifacts across day/week/month/life scales,
- assembling model-facing context packets for interpretation,
- turning successful ad hoc analysis into reusable modules or artifacts.

## Canonical Position

Treat current report surfaces such as `calendar_*` and `life_timeline*`
as delivery surfaces, not as architectural standards.

The real build target is:

1. raw source facts,
2. deterministic features,
3. trajectory artifacts,
4. context packets/state,
5. interpretation and revision.

## Durable Outputs

The skill should bias toward creating or improving:

- reusable `lynchpin.*` modules,
- warehouse-backed typed artifacts,
- context-packet builders,
- durable methodology notes,
- project-run notes under `project-runs/`.

It should bias away from:

- disposable reports,
- prompt-only interpretation with no reusable substrate,
- app-name-only classification,
- one-off dashboards with no lasting analytical leverage.
